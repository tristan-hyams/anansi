## Context

Continuing from session 06 (weaver + CLI wiring). This session completed the CLI wiring, added comprehensive output files, stats, sitemap tree, and did a full documentation audit.

## Work Done

### Phase 6 - CLI Wiring

- `main.go` wires `ParseFlags → OriginURL → NewWeaver → Weave → writeOutputFiles`
- Exit codes: 0 success, 1 error, 130 SIGINT
- Terminal shows JSON logs (stderr) + one-liner completion summary
- `make run ARGS="-workers 5 -rate 10 -max-depth 3"` support added to Makefile

### Output Files (auto-generated every crawl)

- **`crawl-results.md`** - spider banner, latency stats (avg/P50/P95/P99/min/max), status code distribution, content-type breakdown, flat page list sorted by depth, directory tree sitemap
- **`crawl-results.json`** - machine-readable JSON with same data, human-readable durations, pipeable to `jq`
- **`crawl-errors.md`** - errors grouped by reason, each URL with RFC3339 timestamp, count per group

### Stats & Analytics

- `ComputeStats()` - aggregates from `PageResult` data
- Latency percentiles: avg, P50, P95, P99, min, max
- Status code distribution (200, 301, 404, 5xx counts)
- Content-Type breakdown (text/html, application/json, etc.)
- `ContentType` field added to `PageResult`, set by crawler on every response

### Sitemap Tree

- Directory tree built from URL paths using trie
- Box-drawing characters (├── └── │) for visual hierarchy
- Skipped pages excluded from tree
- Shows actual directory structure vs hop-based depth

### Naming Finalization

- `seed` → `origin` across entire codebase (normalizer, config, weaver, tests, docs)
- Defaults: workers=1, rate=1, max-depth=1 (conservative)

### Bug Fixes

- **Max depth off-by-one:** `>=` changed to `>`. MaxDepth=3 now crawls depths 0,1,2,3.
- **Skipped pages now recorded:** depth-exceeded pages recorded with `errMaxDepth` error, appear in skipped count and error log.
- **Frontier pending counter:** deterministic completion via `IsDone()` - `pending.Load() <= 0 && len(queue) == 0`. Replaces the racy `active` counter approach.

### Docker Improvements

- `CMD ["https://crawlme.monzo.com/"]` added - image works standalone
- `ENTRYPOINT ["anansi"]` + `CMD` pattern: `docker run --rm anansi -workers 5 https://example.com/`

## Known Issues (documented, not blocking)

### High Priority
- **Goroutine leak potential:** `monitorCompletion` polling - should be safe due to `defer crawlCancel()` in `Weave()` but worth verifying under edge cases
- **Silent link extraction errors:** parser failures logged at Warn but page recorded as success - should store error on PageResult
- **Unused `Status`/`Err` fields on FrontierURL:** defined but never set by the crawl loop - remove or implement

### Medium Priority
- **webutil has 0% test coverage:** singleton Transport pattern untested
- **`fmt.Sprintf` in frontier debug log:** allocates even when debug logging disabled - use structured args instead
- **`pageURL` computed twice:** once in `processURL`, again in `handleResponse` - pass as parameter

### Low Priority
- **Repeated sort-map-keys pattern:** in stats/result/error formatting - extract generic helper
- **50ms polling interval hardcoded:** `monitorCompletion` poll rate - move to constant
- **`WeaverConfig.BufferSize` unused:** field exists but always passes 0 to frontier (which uses its own default)
