## Context

## Work Done

### Phase 4 - robots

- **robots.go** - `Fetch()` retrieves and parses robots.txt. `Rules` wrapper with `IsAllowed(path)` and `CrawlDelay()`.
  - Graceful degradation: 404 and 403 return allow-all, 5xx/other 4xx return error, network error returns allow-all with warning.
  - Request sets `User-Agent: Anansi` and `Accept: text/plain` headers.
  - Uses `github.com/temoto/robotstxt` for parsing.
  - Refactored to `handleResponse` and `parseBody` helpers to keep cyclomatic complexity under revive's limit.

- **xrobotstag.go** - `ParseXRobotsTag(header)` parses the `X-Robots-Tag` HTTP header. Returns `Directives` struct with `NoIndex`, `NoFollow`, and `ShouldFollow()` method.
  - Per-page check - the crawler calls this on every response before extracting links.
  - If `nofollow` or `none` is set, skip link extraction.

- **87.9% coverage**, 15 unit tests + 2 live integration tests.

### Design Decisions

#### 403 treated as not found
CDN and static hosting providers (S3 behind CloudFront, Azure Blob Storage, GCS) return 403 for missing files instead of 404. This happens because the bucket doesn't allow public listing - the server can't distinguish "not found" from "no access" without revealing bucket contents.

Initially implemented a HEAD probe to disambiguate, but analysis showed HEAD returns the same status as GET on these hosts - the probe doesn't add information. Simplified to treating 403 as not found for robots.txt.

Verified live against crawlme.monzo.com: both `/robots.txt` and `/definitely-does-not-exist.html` return 403 while `/` returns 200.

#### X-Robots-Tag discovery
crawlme.monzo.com returns `X-Robots-Tag: noindex,follow` on page responses. This was not in the original PLAN.md or ARCH.md - discovered during the robots.txt investigation. Added as a per-response check complementing the one-time robots.txt fetch.

The X-Robots-Tag check happens on the GET response the crawler already makes - no additional HTTP request needed.

#### Status code semantics
- **404:** file doesn't exist. No rules. Allow all.
- **403:** on robots.txt, treated as not found (CDN behavior). Allow all.
- **5xx:** server error. File may exist. Return error to caller.
- **Other 4xx:** return error to caller.

### ARCH.md Pipeline Update
Updated the page fetch pipeline to include X-Robots-Tag check between Content-Type check and HTML parsing. The full flow is now: dequeue → rate limit → GET → Content-Type check → X-Robots-Tag check → parse → normalize/filter → enqueue.
