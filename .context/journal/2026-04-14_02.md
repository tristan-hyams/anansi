## Context

Continuing from session 01 (design scaffolding). This session initialised the Go module and built the CLI entry point with config, logging, and tooling.

## Work Done

### Phase 0 - Project Init
- `go mod init github.com/tristan-hyams/anansi`

### CLI Entry Point (cmd/anansi)
- **config.go** - `AnansiConfig` struct with JSON tags for serialization. `ParseFlags()` registers CLI flags and returns validated config. `SeedURL()` validates the seed. `SlogLevel()` maps string to `slog.Level`. `SaveToFile()`/`LoadConfigFromFile()` for JSON config persistence.
- **consts.go** - Default constants extracted to dedicated file (workers=5, rate=5.0, timeout=30s).
- **logger.go** - `setupLogger()` creates slog JSON handler to stderr, respects config log level, sets as default.
- **main.go** - Thin wiring: ParseFlags → setupLogger → setupSignalContext. Only place that calls `os.Exit`.
- **config_test.go** - `package main_test` with testify assert/require. Tests for SeedURL validation, SlogLevel mapping, SaveToFile/LoadConfigFromFile round-trip, error paths.

### Tooling
- **revive.toml** - `enableAllRules = true` with surgical overrides. Test files excluded from add-constant, cognitive-complexity, unhandled-error, unused-parameter, function-length.
- **anansi.code-workspace** - VS Code workspace with F5 debug launch config, revive lint-on-save.
- **Makefile** - Windows-compatible (no bash dependency). Uses `cmd` native commands for clean.

## Coding Rules Established

- Only `main()` may call `os.Exit` or `log.Fatal` - helpers return errors.
- Return `(nil, err)` with pointer types, not zero-value structs.
- Constants go in `consts.go`, not inline.
- Always verify with `make build`, `make test`, `make lint`.
- Revive lint warnings should be addressed, not silenced - if a rule fires, fix the code or understand why.

## Makefile Windows Compatibility

GNU Make from Chocolatey on Windows doesn't honour `SHELL := bash` or `SHELL := C:/PROGRA~1/Git/bin/bash.exe`. Resolved by using Windows-native commands (`if exist bin rmdir /s /q bin` instead of `rm -rf bin/`).

## Dependencies Added

## Lint Status

One remaining warning: `deep-exit` on `flag.Parse()` inside `ParseFlags()`. This is a known trade-off - `flag.Parse` can call `os.Exit(2)` on parse errors. Accepted as-is since ParseFlags is only called from main.
