# Session 03 - Phase 1 (normalizer) + Phase 2 (parser)

**Date:** 2026-04-14

## Context

Continuing from session 02 (Phase 0 init + CLI). This session built the two leaf packages that have no internal dependencies: URL normalizer and HTML parser.

## Work Done

### Phase 1 - normalizer

- **normalizer.go** - three pure functions:
  - `Normalize(base, raw)` - resolves relative URLs, strips fragments, lowercases scheme/host, strips default ports. Trailing slashes intentionally NOT normalized per RFC 3986 Section 6 - /path and /path/ are distinct URIs.
  - `IsSameHost(seed, candidate)` - strict hostname match. Renamed from `IsSameDomain` to accurately reflect the spec's "limited to one subdomain" requirement. `crawlme.monzo.com` does not match `community.monzo.com` or `monzo.com`.
  - `IsFollowableScheme(u)` - only http/https.
- **stripDefaultPort** uses `net.SplitHostPort` instead of string suffix matching to avoid ambiguity (e.g. `:8080` being confused with `:80`).
- **95% coverage**, 20+ table-driven test cases.

### Phase 2 - parser

- **parser.go** - `ExtractLinks(ctx context.Context, r io.Reader) ([]string, error)`
  - Uses `golang.org/x/net/html` tokenizer (state machine - sequential, no parallelism within a single page parse).
  - Handles both `StartTagToken` and `SelfClosingTagToken` for `<a/>` edge case.
  - `isAHrefToken` - fast byte check: `len(name) == 1 && name[0] == 'a'`. Also guards against `<area>` (4 bytes, fails `len == 1`).
  - `getHrefValue` - uses `bytes.Equal(key, hrefKey)` with package-level `var hrefKey = []byte("href")` to avoid string allocation on every attribute comparison.
  - Context checked each iteration for cancellation of large/buffered documents.
  - Returns raw hrefs unfiltered - normalization/filtering is the caller's responsibility.
- **95.7% coverage**, 9 unit tests + 1 live integration test.
- **Testdata fixtures:** well_formed.html, malformed.html, no_links.html, mixed_tags.html.

## Design Decisions

- **Trailing slash normalization (Option D):** Don't normalize. Per RFC 3986 Section 6, `/path` and `/path/` are distinct URIs. Content may differ - the origin server decides. Let the crawler treat them as distinct and rely on HTTP redirects + dedup for equivalence.
- **`IsSameDomain` → `IsSameHost` rename:** The spec says "limited to one subdomain". The function compares hostnames, not domains. Name now matches intent.
- **Context on ExtractLinks:** Tokenizer is a state machine reading sequentially. If the body is buffered in memory, HTTP timeout won't help. Context check per iteration gives the caller a cancellation path.
- **`findProjectRoot` uses `.git` not `go.mod`:** Mono-repos with multiple `go.mod` files would stop at the wrong level.
