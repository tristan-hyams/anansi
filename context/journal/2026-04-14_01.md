# Session 01 — Project Design & Scaffolding

**Date:** 2026-04-14

---

## Context

Anansi is a take-home challenge: build a simple web crawler in Go. Given a starting URL, visit every page on the same subdomain, print URLs visited and links found per page.

## Design Decisions Made

### Concurrency
- Worker pool with configurable concurrency, not goroutine-per-URL.
- Buffered channel as work queue with natural backpressure.
- `sync.WaitGroup` for natural completion detection.
- Global rate limiter (`golang.org/x/time/rate` token bucket) — not per-worker backoff.
- `signal.NotifyContext` for graceful SIGINT/SIGTERM shutdown.

### Frontier / Dedup
- `Frontier` interface with in-memory implementation (seen-set + buffered channel).
- Production swap to Redis/RabbitMQ via interface — mentioned in README, not implemented.
- `sync.Map` for visited URL tracking.

### URL Handling
- Normalize: strip fragments, lowercase scheme+host, resolve relative URLs, strip default ports.
- Scheme filter: only follow `http` and `https`. Log and skip `mailto:`, `javascript:`, `tel:`.
- Domain filter: strict hostname match (no `www.` equivalence).
- Trailing slash: pick one convention and be consistent.

### HTML Parsing
- `golang.org/x/net/html` tokenizer (not full DOM walk). Only extracting `<a href>` links.
- SPA/JS-rendered content explicitly out of scope — documented in README.

### Content-Type
- Check `Content-Type: text/html` before parsing. Skip non-HTML (PDF, images, JSON).

### robots.txt
- Fetch once at crawl start. Respect `User-agent: *` Disallow rules.
- `github.com/temoto/robotstxt` for parsing.

### Output
- Structured logging via `slog` during crawl.
- Summary report to stdout on completion (flat URL list with link counts).

### Testing
- Table-driven unit tests for normalizer, parser, robots.
- `httptest.NewServer` integration tests with canned HTML in `testdata/`.
- `-race -cover` always on.

### Tooling
- Makefile: build, test, lint, run, clean, docker, docker-run.
- Multi-stage Dockerfile (golang:1.24-alpine → alpine) for reviewer convenience.
- `golangci-lint` for static analysis.

### Project Structure
- Single module, top-level packages (no `internal/`). README notes where `go.work` multi-module would apply.
- Packages: `cmd/anansi`, `crawler`, `frontier`, `parser`, `normalizer`, `robots`.
